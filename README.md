Este repositorio contiene una **aplicación demostrativa de Spring Boot que ilustra la gestión de transacciones ACID en un sistema de e-commerce**. La aplicación implementa un flujo transaccional donde se procesan órdenes de compra junto con sus pagos asociados: cuando un cliente envía una solicitud para realizar un pedido (orden + pago), el sistema utiliza `@Transactional` para garantizar que ambas operaciones se completen exitosamente o se reviertan por completo en caso de error. Usa Spring Boot 2.7.1 con Java 11, Spring Data JPA/Hibernate para el acceso a datos, MySQL 8 como base de datos (gestionada con Docker Compose), y expone un endpoint REST para crear órdenes. El caso de uso principal demuestra cómo las transacciones mantienen la integridad de los datos: si el pago es rechazado (por ejemplo, si se intenta usar una tarjeta de débito), la orden tampoco se guarda, evitando inconsistencias entre las tablas `orders` y `payments`.
